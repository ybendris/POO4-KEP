Le checker peut être utilisé de 2 façons différentes.

1)	fichierInstance
	
	Dans ce cas, le fichier instance et le fichier solution doivent être dans le même répertoire 
	que le checker et le fichier solution doit être nommé "fichierInstance_sol.txt".

	
2)	-all
	
	Dans ce cas, toutes les solutions dans le répertoire du checker sont testées. 
	Tous les fichiers solutions doivent terminer par "_sol.txt".
	Les fichiers instances doivent être dans le même répertoire et nommés "fichierInstance.txt" pour être associés à "fichierInstance_sol.txt".
